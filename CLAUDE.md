## Claude Code specifics

### Skills

Native skill files live in `~/.claude/skills/<name>/SKILL.md` (user)
and `.claude/skills/<name>/SKILL.md` (project, generated by
`ai-eng sync-mirrors`).

The single source of truth is `skills/catalog/<name>/SKILL.md`. Edit
there, run `ai-eng sync-mirrors`, and commit both.

### Hooks

Hooks live in `.claude/settings.json` under `hooks.{PreToolUse, PostToolUse,
SessionStart, Stop, UserPromptSubmit, PreCompact}`. They are
hash-verified. To modify, change the source script in
`python/ai_eng_hooks/<name>.py` and run `ai-eng doctor --fix`.

### Plugins

Use `/plugin marketplace add @ai-engineering/official-marketplace` to
register the official tier. Plugin trust verification (Sigstore + SLSA
+ SBOM + Scorecard) happens automatically on install.

### Subagents

Available agents: `builder`, `planner`, `verifier`, `reviewer`,
`explorer`, `orchestrator`, `security-scanner`. The `builder` is the
ONLY agent with write permissions. See
[ADR-0001](./docs/adr/0001-hexagonal-architecture.md).

## Token efficiency

- Don't dump generated mirror files into context — they're regenerated
  from `skills/catalog/`.
- Use `/clear` between unrelated tasks.
- Heavy exploration → dispatch `explorer` agent (read-only, isolated
  context).

## Hot path discipline

Pre-commit hooks must complete in < 1 second. If you find a hook taking
longer, it's a bug — file it under `infra:slow-hook`.
