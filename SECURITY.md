# Security Policy

## Supported versions

This is an **alpha** release. Only the latest tagged version is
supported. Once we hit a stable release we will publish a support
window.

## Reporting a vulnerability

Please **do not** open public issues for security findings. Use one of
these channels:

1. **GitHub private security advisories** — https://github.com/soydachi/ai-engineering/security/advisories/new
2. **Email** — security@ai-engineering.dev (PGP key in
   `docs/pgp-key.asc` once published)

Please include:

- A description of the issue and its impact.
- Steps to reproduce, or a proof-of-concept.
- Affected version(s) / commit hash.
- Suggested mitigation, if you have one.

## Disclosure timeline

- We acknowledge within **72 hours**.
- We commit to a triage outcome within **7 days**.
- Critical: patch within **15 days** (matches our internal
  risk-acceptance TTL ladder).
- High: patch within **30 days**.
- Medium: patch within **60 days**.
- Low: patch within **90 days**.

We coordinate the disclosure date with the reporter once a fix is
ready.

## Supply chain

- Releases are signed via Sigstore keyless OIDC.
- SBOM (CycloneDX 1.6) is published with every release.
- SLSA v1.0 provenance is generated by the
  `slsa-framework/slsa-github-generator` reusable workflow pinned to
  an immutable commit SHA.
- We monitor Shai-Hulud-style worm patterns continuously and enforce
  `--ignore-scripts` in our own CI.

## Scope

In-scope:

- The framework itself (`packages/`, `python/`, `skills/`, `agents/`).
- Plugins published in `@ai-engineering/*` (OFFICIAL tier).

Out-of-scope:

- Plugins in COMMUNITY or VERIFIED tiers — report directly to those
  authors. We can de-list compromised plugins at the registry layer.
- IDE hosts (Claude Code, Cursor, Codex CLI, …) — report to their
  respective vendors.
- LLM providers (Anthropic, OpenAI, Google, …) — report to vendors.

## Bug bounty

We do not run a paid bounty programme yet. We do publish a public
acknowledgements page (`SECURITY-THANKS.md`) for confirmed reports
that follow this policy.
