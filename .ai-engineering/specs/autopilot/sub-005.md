# Sub-Spec 005: Service Refactor + Update Migration

## Scope

Refactor `installer/service.py` to orchestrate the phase pipeline instead of inline logic. Migrate `updater/service.py` to reuse the same phases with overwrite semantics. Fix the vcs_provider bug and already_installed detection as part of the refactor. Preserve the existing backup/rollback mechanism from the updater.

## Work Units

- W-018: Rewrite `installer/service.py::install()` to: (1) build an `InstallContext` from arguments, (2) create the 6 phases in order, (3) run them through `PipelineRunner`, (4) return `InstallResult` derived from `PipelineSummary`. The vcs_provider parameter is now properly passed through InstallContext to all phases. The already_installed detection uses `install-manifest.json` existence.
- W-019: Add install modes to `InstallContext`: `fresh` (overwrite framework), `repair` (create-only), `reconfigure` (add/remove providers), `install` (first-time). Each phase reads the mode to determine behavior.
- W-020: Migrate `updater/service.py` to use the phase pipeline. The update function: (1) runs all phases in dry-run mode to produce a plan, (2) shows the plan as diff, (3) on `--apply`, executes the plan with overwrite semantics for framework-owned files. Preserve the existing backup/rollback mechanism.
- W-021: Ensure the update CLI interface (`ai-eng update` shows diff, `ai-eng update --apply` writes) is unchanged from the caller's perspective.
