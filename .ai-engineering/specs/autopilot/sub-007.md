# Sub-Spec 007: Tests + CI

## Scope

Write comprehensive tests for all new code: unit tests per phase, integration test matrix for provider/VCS combinations, re-install scenario tests, settings.json merge tests, backward compatibility tests for the update command, and update the CI smoke workflow.

## Work Units

- W-027: Unit tests for each phase — `test_phases.py` tests plan()/execute()/verify() for governance, ide_config, hooks, state, tools phases using fixture directories (tmp_path). Each phase tested in isolation.
- W-028: Integration test matrix — `test_install_matrix.py` tests 10 combinations: 8 single-provider (claude_code/github_copilot/gemini/codex x github/azure_devops) + 2 multi-provider ([claude_code, github_copilot] x github, [claude_code, gemini] x github). Verifies correct files created per combination.
- W-029: Merge + re-install + failure tests — `test_merge.py` tests settings.json merge scenarios (add hooks, preserve user hooks, preserve permissions, malformed fallback). Re-install tests cover fresh/repair/reconfigure paths including provider add/remove. Phase failure resilience: hooks phase failure does not prevent prior phases from completing.
- W-030: Update `install-smoke.yml` to test the new wizard with `--non-interactive` on ubuntu, macos, windows. Add a dry-run validation step.
- W-031: Additional tests — `test_detect.py` for auto-detection (VCS, tools, auth, legacy migration). `test_pipeline.py` for PipelineRunner (dry-run, failure handling, plan serialization). `test_ui.py` for rich fallback behavior. Windows PS1 syntax validation. Backward compat test for update command.

- [ ] Re-install tests cover fresh (overwrite framework, preserve team/system), repair (create-only), reconfigure (add/remove provider)
