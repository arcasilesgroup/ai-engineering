# Sub-Spec 003: Phase Pipeline Foundation

## Scope

Create the phase pipeline infrastructure: the PhaseProtocol interface, data models (PhasePlan, PhaseResult, PhaseVerdict), the pipeline runner, and the first phase (detect). This establishes the architecture that all subsequent phases implement.

## Work Units

- W-009: Define `PhaseProtocol` (runtime-checkable Protocol class) with three methods: `plan(context: InstallContext) -> PhasePlan`, `execute(plan: PhasePlan) -> PhaseResult`, `verify(result: PhaseResult) -> PhaseVerdict`. Define `InstallContext` dataclass holding target path, providers, vcs_provider, stacks, ides, mode (install/repair/fresh/reconfigure), existing_manifest (optional).
- W-010: Define `PhasePlan` dataclass: list of `PlannedAction` items, each with type (create/overwrite/merge/skip), source (relative), destination (relative), rationale. JSON-serializable via `to_dict()`/`from_dict()`. Path security: all destinations MUST be relative, reject absolute paths and `../` traversal.
- W-011: Define `PhaseResult` (created/skipped/failed file lists) and `PhaseVerdict` (passed: bool, warnings: list, errors: list).
- W-012: Create `pipeline.py` — `PipelineRunner` class that takes a list of phases, runs plan->execute->verify for each sequentially, handles phase failure (skip remaining, record in summary), produces a `PipelineSummary` with per-phase verdicts. Supports dry-run mode (plan only, no execute/verify).

## Acceptance Criteria

- [ ] `PhaseProtocol` is a `typing.Protocol` with `plan`, `execute`, `verify` methods
- [ ] `PhasePlan.to_dict()` produces valid JSON; `PhasePlan.from_dict()` round-trips correctly
- [ ] `PhasePlan` rejects actions with absolute destination paths (raises ValueError)
- [ ] `PhasePlan` rejects actions with `../` in destination paths (raises ValueError)
- [ ] `PipelineRunner` in dry-run mode calls `plan()` on all phases but never calls `execute()`
- [ ] `PipelineRunner` skips remaining phases when one fails, records failure in summary
- [ ] `DetectPhase.plan()` auto-detects VCS from git remote, detects installed tools via shutil.which, detects auth status
- [ ] `DetectPhase` detects legacy `context/` path and plans migration to `contexts/`
