# Sub-Spec 003: Install Pipeline E2E Tests

## Scope

Create `tests/e2e/test_install_pipeline.py` with >= 3 tests covering `install_with_pipeline()`.

## Exploration

- `install_with_pipeline(target, mode, stacks, ides, vcs_provider, ai_providers, dry_run)` returns `tuple[InstallResult, PipelineSummary]`
- `dry_run=True` skips execute+verify, returns plans only, no files written
- Mode auto-detection: if `install-state.json` exists, promotes INSTALL -> REPAIR
- Operational phases (tools, VCS auth, branch policy) need stubs via monkeypatch
- Stub pattern from test_install_operational_flows.py: `_StubProvider(available=False)` short-circuits network
- `PHASE_ORDER` has 6 phases: detect, governance, ide_config, state, hooks, tools

## Plan

### Tests (4 total)

1. `test_pipeline_dry_run_no_files_written` — dry_run=True, verify no .ai-engineering dir created, summary.plans has 6 entries, summary.results is empty
2. `test_pipeline_clean_install_creates_governance` — full run with stubbed ops, verify .ai-engineering/ structure, result.total_created > 0, all 6 phases completed
3. `test_pipeline_auto_detects_repair_mode` — call twice, second auto-promotes to REPAIR, result.already_installed is True
4. `test_pipeline_summary_result_mapping` — verify result.governance_files.created correspond to real files on disk

### Stub pattern:
```python
monkeypatch.setattr(service, "check_tools_for_stacks", lambda s: SimpleNamespace(tools=[]))
monkeypatch.setattr(service, "ensure_tool", lambda t: SimpleNamespace(available=True, detail="ok"))
monkeypatch.setattr(service, "get_provider", lambda p: _StubProvider(available=False))
monkeypatch.setattr(service, "provider_required_tools", lambda v: [])
```
