# Sub-Spec 001: Template Parity

## Scope

Close all gaps between the template directory and the live dogfooding project. After this sub-spec, the template ships every hook script, the complete settings.json, correct naming conventions, and no dead paths. This is the foundation — all other sub-specs depend on having the right files in the template.

## Work Units

- W-001: Copy ALL hook scripts from `scripts/hooks/` to `templates/project/scripts/hooks/`. Include the full `_lib/` directory, all Python hooks (auto-format.py, cost-tracker.py, instinct-extract.py, mcp-health.py, observe.py, prompt-injection-guard.py, strategic-compact.py, telemetry-skill.py), and all Copilot shell hooks (copilot-*.sh, copilot-*.ps1). Template count MUST match `find scripts/hooks -type f -not -path '*__pycache__*' | wc -l`.
- W-002: Replace `templates/project/.claude/settings.json` with the full live configuration from `.claude/settings.json`. Ensure all 7 hook groups are present (UserPromptSubmit, PreToolUse, PostToolUse, PostToolUseFailure, Stop). Paths must use `$CLAUDE_PROJECT_DIR` for portability.
- W-003: Rename `templates/.ai-engineering/context/` to `templates/.ai-engineering/contexts/` (plural). Move the 2 product contract files. Update any internal cross-references in template files.
- W-004: Delete the orphaned `templates/project/.ai-engineering/` directory (dead path — no install code processes it).
- W-005: Verify `templates/project/instructions/` is complete and matches what `_PROVIDER_TREE_MAPS["github_copilot"]` expects.
