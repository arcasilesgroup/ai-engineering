# Sub-Spec 001: VCS CLI Tests

## Scope

Create `tests/unit/test_vcs_cmd.py` with >= 7 tests covering `vcs_status` and `vcs_set_primary` CLI functions.

## Exploration

- `vcs_status(target)`: resolve root, check `.ai-engineering` dir, load manifest, get provider, display
- `vcs_set_primary(provider_name, target)`: validate against `("github", "azure_devops", "azdo")`, resolve alias, update manifest
- JSON mode: `emit_success/emit_error` for structured output, `kv/error/success` for human
- Error codes: `NO_FRAMEWORK`, `INVALID_PROVIDER`
- Pattern from test_provider_cli.py: CliRunner + monkeypatch fixture dict, `_FakeManifest` class
