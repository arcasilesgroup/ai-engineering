# Sub-Spec 004: Core Phases + settings.json Merge

## Scope

Implement the 5 remaining phases (governance, ide_config, hooks, state, tools) and the settings.json intelligent merge module. Each phase follows the PhaseProtocol from sub-003.

## Work Units

- W-013: `GovernancePhase` — plans and copies `.ai-engineering/` template tree. Respects ownership: excludes `contexts/team/**` and `state/**` paths. In fresh mode, overwrites framework-owned governance files. In repair mode, creates only missing files.
- W-014: `IdeConfigPhase` — plans and copies IDE-specific files based on providers. Uses `resolve_template_maps()` from sub-002. Handles VCS-specific templates. In reconfigure mode, uses `remove_provider_templates()` for removed providers.
- W-015: `HooksPhase` — copies hook scripts to `scripts/hooks/`, installs git hooks via `hooks/manager.py`, and merges `.claude/settings.json`. Delegates to `merge.py` for the merge logic.
- W-016: `StatePhase` — generates or updates state files. In fresh mode, regenerates `install-manifest.json` and `ownership-map.json`. In all modes, preserves `audit-log.ndjson` and `decision-store.json` (append-only).
- W-017: `ToolsPhase` — verifies/installs required tools, checks VCS auth, records readiness in manifest. Reports unresolved tools as warnings (not failures) for delegation to `ai-eng setup`.

## Acceptance Criteria

- [ ] `GovernancePhase` in fresh mode overwrites all `.ai-engineering/` files except `contexts/team/**` and `state/**`
- [ ] `GovernancePhase` in repair mode creates only missing files
- [ ] `IdeConfigPhase` with providers `[claude_code, github_copilot]` copies both Claude and Copilot files
- [ ] `IdeConfigPhase` reconfigure from `[claude_code, copilot]` to `[claude_code]` removes Copilot files, preserves shared AGENTS.md if needed
- [ ] `HooksPhase` copies all hook scripts and calls `install_hooks()` for git hooks
- [ ] `merge.py` adds missing hooks from template without removing user-added hooks
- [ ] `merge.py` adds missing deny rules without removing user-added permission rules
- [ ] `merge.py` preserves user-added top-level keys
- [ ] `merge.py` falls back to template copy when destination is malformed JSON
- [ ] `StatePhase` in fresh mode regenerates install-manifest.json and ownership-map.json
- [ ] `StatePhase` never overwrites audit-log.ndjson or decision-store.json
- [ ] `ToolsPhase` reports missing tools as warnings, not errors
