# Sub-Spec 006: Wizard UX + CLI Modes

## Scope

Implement the guided wizard experience in the CLI: numbered progress steps, auto-detection with confirmation, re-install detection with 4 options, summary panel with next actions, and the --dry-run/--plan/--non-interactive modes.

## Work Units

- W-022: Create `installer/ui.py` — Rich-based rendering module with: `render_step(n, total, name, description, status, detail)`, `render_summary(pipeline_summary)`, `render_reinstall_options()`, `render_detection_results(detect_result)`. Falls back to plain text if `rich` is unavailable.
- W-023: Add re-install detection to `install_cmd()` in core.py: check for `install-manifest.json`, if found show 4 options (fresh/repair/reconfigure/cancel) using `ui.render_reinstall_options()`. Map selection to `InstallContext.mode`.
- W-024: Add auto-detection flow to `install_cmd()`: run DetectPhase first, show results via `ui.render_detection_results()`, confirm with user (max 3 questions: providers, VCS, stacks). Delegate undetectable config to `ai-eng setup` in the summary.
- W-025: Add interactive prompts for stacks and IDEs when `--stack`/`--ide` flags are absent. Use Typer prompt with defaults from auto-detection.
- W-026: Add CLI flags: `--dry-run` (output JSON plan to stdout, no files), `--plan plan.json` (replay saved plan with path security validation), `--non-interactive` (auto-detected values + defaults, no prompts).

- [ ] Install progress shows `[N/6]` numbered steps with description, spinner, checkmark/warning
