# Sub-Spec 002: Templates Public API + VCS Fix

## Scope

Extract a public API from templates.py that both installer and updater can consume, eliminating private import coupling. Fix the VCS template wiring so GitHub templates (CODEOWNERS, dependabot.yml, PR template) are actually deployed.

## Work Units

- W-006: Create `resolve_template_maps(providers, vcs_provider)` public function that returns the complete set of file maps, tree maps, common maps, and VCS maps for a given configuration. This replaces direct access to `_PROVIDER_FILE_MAPS`, `_PROVIDER_TREE_MAPS`, `_COMMON_FILE_MAPS`, `_COMMON_TREE_MAPS`, and `_VCS_TEMPLATE_TREES`.
- W-007: Ensure `copy_project_templates()` properly handles `vcs_provider` parameter — the function signature already supports it, but callers must pass it. Verify the `_VCS_TEMPLATE_TREES` routing works end-to-end.
- W-008: Remove legacy `_PROJECT_TEMPLATE_MAP` and `_PROJECT_TEMPLATE_TREES` aggregated maps. Updater imports will be migrated in sub-005.
