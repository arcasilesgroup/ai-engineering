# Sub-Spec 002: Release Flow Fix

## Scope

When `config.wait` is False, skip the tag phase (not just wait-for-merge) to prevent creating tags on unmerged commits. Output a clear message directing users to re-run with `--wait` after manual merge. Update release tests for new skip behavior.

## Exploration

- Bug at orchestrator.py:221: `_create_tag()` called unconditionally after the wait-for-merge skip
- `_create_tag()` runs `git checkout main && git pull --ff-only` — tags wrong commit if PR not merged
- Without `--wait`, also need to skip: tag, manifest update, deploy event, monitor
- Idempotent resume works: re-run with `--wait` finds existing PR, waits for merge, then tags correctly
- No existing test covers `execute_release()` with `wait=False`

## Plan

### Step 1: Restructure execute_release() in orchestrator.py (lines 210-265)
- Move tag, manifest, emit_deploy_event, and monitor phases INSIDE `if config.wait:` block
- `else` block emits 4 skipped PhaseResults:
  - wait-for-merge: `"--wait off"`
  - tag: `"Tag creation deferred -- run ai-eng release <version> --wait to tag after merge"`
  - manifest: `"--wait off"`
  - monitor: `"--wait off"`
- `result.success = True` falls through for both paths

### Step 2: Add 2 new tests in test_release_orchestrator.py
- Test A: `test_execute_release_no_wait_skips_tag_and_returns_success` — verify phases include skipped tag, no `_create_tag` called
- Test B: `test_execute_release_no_wait_then_wait_resumes` — simulate two calls, verify resume path works

### Step 3: Verify
