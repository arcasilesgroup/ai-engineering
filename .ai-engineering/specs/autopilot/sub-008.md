# Sub-Spec 008: Review Remediation

## Problem

A comprehensive 4-agent code review (correctness, testing, security, architecture) of the spec-064 branch identified 14 actionable findings: 7 blocking (bugs + security), 7 quality/hardening. These must be resolved before the PR.

## Scope

### In Scope

**A) Critical Bugs (7 blocking)**

1. **RECONFIGURE dead code**: `install_with_pipeline()` never populates `existing_manifest` in `InstallContext`. `IdeConfigPhase` gates RECONFIGURE cleanup on `context.existing_manifest` which is always `None`. Provider removal is silently skipped.
   - Fix: Load existing manifest via `read_json_model(path, InstallManifest)` when mode is REPAIR or RECONFIGURE. If the manifest is malformed or unreadable, log a warning and proceed with `existing_manifest=None` (degrading RECONFIGURE to INSTALL-like behavior). MUST NOT crash.

2. **Path inconsistency in ide_config._copy_tree**: Stores absolute paths in `result.created` while all other phases store relative paths. Breaks any consumer comparing paths.
   - Fix: Compute relative paths via `.relative_to(context.target)` in `_copy_tree`, matching hooks.py pattern.

3. **Deletions recorded as "created"**: `ide_config.execute()` appends `"deleted:{path}"` strings to `result.created`. Inflates `total_created` count and confuses path-based consumers.
   - Fix: Add `deleted: list[str]` field to `PhaseResult`. Record deletions there. Update `_summary_to_install_result` and `_render_pipeline_steps` to account for the new field. Dry-run JSON output MUST include deletions. CLI MUST display deletion count separately.

4. **Source path traversal in plan deserialization**: `PlannedAction.__post_init__` validates `destination` for `../` and absolute paths but applies zero validation to `source`. Crafted `plan.json` loaded via `PhasePlan.from_dict()` can exfiltrate arbitrary files.
   - Fix: Apply identical `../` and absolute path validation to `source` in `PlannedAction.__post_init__` (lines 47-53).

5. **JSON injection in hook shell printf fallbacks**: Three shell hooks use `printf` with incomplete or missing escaping:
   - `telemetry-skill.sh` (line 40-41): raw `printf` with NO escaping. Line 46 writes `$PROMPT` (full user input) directly into printf -- highest risk.
   - `telemetry-agent.sh` (line 71-72): `tr '"' "'"` misses backslashes.
   - `copilot-agent.sh` (line 94-95): has safe `jq` path, but `printf` fallback has same incomplete escaping.
   - Fix: Add `safe_json_string()` shell function in all 3 files that escapes `\`, `"`, `\n`, `\r`, `\t` before `printf`.

6. **Two parallel install paths**: `install()` legacy calls `_run_operational_phases()` (VCS auth, branch policy, tooling readiness). `install_with_pipeline()` skips all of this. Tests validate legacy path, CLI uses pipeline path.
   - Fix: `install_with_pipeline()` MUST call `_run_operational_phases()` as a post-pipeline step, reusing the existing function. Remove the bare `except Exception: pass` at line 242. ToolsPhase migration is deferred.

7. **FRESH-mode tree copy duplicated in 2 phases**: `ide_config._copy_tree` (lines 44-56) and `hooks.py` (lines 68-83) implement the same FRESH vs non-FRESH branching. (`governance.py` uses action-type dispatch, not the same pattern.)
   - Fix: Extract shared `copy_tree_for_mode()` into `templates.py`. `ide_config` and `hooks` phases call it instead of inline logic.

**B) Quality + Hardening (7 findings)**

8. **`resolve_template_maps()` called 3x per phase lifecycle**: plan(), execute(), verify() each call it independently with identical arguments in both `ide_config.py` (lines 67, 93, 123) and `hooks.py` (lines 36, 66).
   - Fix: Call once in `plan()`, store resolved maps on the phase instance. `execute()` and `verify()` read from instance.

9. **Dead legacy template maps**: `_PROJECT_TEMPLATE_MAP` and `_PROJECT_TEMPLATE_TREES` in `templates.py` (lines 129-141) are marked "kept for updater backward compatibility" but `updater/service.py` already uses `resolve_template_maps()`. 4 test files still reference them.
   - Fix: Verify sub-002 W-008 status first. If already removed, mark as pre-satisfied. If not, update test imports BEFORE removing symbols from `templates.py`. Verify `pytest --collect-only` passes after each step.

10. **DetectPhase mutates context during plan()**: `plan()` sets `context.vcs_provider` as a side effect, violating the dry-run contract (plan is observation-only).
    - Fix: Store detected VCS in `PhasePlan` metadata. Apply mutation in `execute()`.

11. **Ineffective test assertions**: `assert len(tool_actions) >= 0` in `test_detect.py:48` and `assert len(denied) >= 0` in `test_install_existing.py:114` can never fail (any list length is >= 0).
    - Fix: Replace with meaningful assertions (assert specific content or `> 0`).

12. **`merge_settings` replaces malformed JSON without backup**: User deny rules lost silently on corruption.
    - Fix: Write `settings.json.bak` before replacing. Log warning explicitly mentioning potential deny rule loss.

13. **Bare `except Exception: pass`** in `service.py:242` hides real errors during manifest operational readiness update.
    - Fix: Replace with `except (json.JSONDecodeError, OSError, KeyError) as exc:` and add `logger.debug('Operational readiness update skipped: %s', exc)`.

14. **Phase names hard-coded as strings** in `_summary_to_install_result` and `_render_pipeline_steps`.
    - Fix: Define `PHASE_DETECT`, `PHASE_GOVERNANCE`, `PHASE_IDE_CONFIG`, `PHASE_HOOKS`, `PHASE_STATE`, `PHASE_TOOLS` constants in `phases/__init__.py`. Reference them in both locations.

## Risks

| Risk | Mitigation |
|------|-----------|
| Extracting `copy_tree_for_mode` changes subtle behavior in a phase | Unit tests per phase verify identical output before and after extraction |
| Moving VCS mutation from plan() to execute() breaks dry-run context | Verify dry-run still populates correct VCS for subsequent phases via plan metadata |
| Removing legacy template maps breaks test imports | Update test imports BEFORE removing symbols. `pytest --collect-only` gate at each step |
| `safe_json_string()` escaping misses edge cases | Test with payloads containing `\"`, `\\`, `\n`, `\r`, `\t`, and combined sequences |
| Malformed existing manifest crashes RECONFIGURE | Graceful degradation with warning (AC-R1 explicitly covers this) |
| `PhaseResult.deleted` field not propagated to consumers | AC-R3 explicitly requires `_summary_to_install_result`, `_render_pipeline_steps`, and dry-run JSON to account for it |
